# ASI Roadmap

## Phase 1 — Primitive (v0.1)

Goal: Establish minimal cryptographic identity foundation.

- [x] Ed25519 key model
- [x] DID:key identity format
- [x] Publisher bundle signing
- [x] Invocation envelope signing
- [x] Canonical JSON (JCS) requirement
- [x] Node reference SDK
- [ ] Python reference SDK
- [ ] Formal security review

## Phase 2 — Hardening (v0.2)

- Key rotation format
- Revocation list mechanism
- Optional nonce extension
- Transparency log integration

## Phase 3 — Ecosystem Adoption

- OpenClaw skill integration
- Registry verification badges
- Cross-framework SDK ports
- Enterprise audit integration

ASI becomes a neutral identity primitive adopted across:

- OpenClaw
